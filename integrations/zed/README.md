# Crabbox for Zed

Use Zed as the local control surface for Crabbox remote execution.

This extension does **not** run Zed inside a Crabbox and does not use MCP. Every task invokes the installed `crabbox` CLI from the local repository, so provider selection, authentication, synchronization, lease accounting, and cleanup remain owned by Crabbox.

## What it provides

When `.crabbox.yaml` or `.crabbox.yml` is active, Zed's task picker includes:

- **Crabbox: Doctor** — validate local configuration and connectivity.
- **Crabbox: Spawn reusable box** — run `crabbox warmup` and keep the returned lease available.
- **Crabbox: Run selected command** — execute the selected shell command through `crabbox run`.
- **Crabbox: Run detected project job** — run the job generated by `crabbox init --detect`.
- **Crabbox: List boxes** — show current leases.
- Interactive status, remote-command, SSH, inspect, and stop tasks that first show the current lease list and then ask for an id or slug.

The extension also adds Crabbox YAML highlighting and snippets for project configuration, jobs, and cache volumes.

## Requirements

Install the Crabbox CLI and verify it is available to Zed's terminal environment:

```sh
brew install openclaw/tap/crabbox
crabbox --version
```

Initialize the repository if needed:

```sh
crabbox init --detect
```

## Install as a development extension

Until the extension is published in Zed's extension registry:

1. Clone `openclaw/crabbox`.
2. In Zed, run **zed: extensions**.
3. Choose **Install Dev Extension**.
4. Select `integrations/zed` from the checkout.
5. Open the target repository's `.crabbox.yaml` and run **task: spawn**.

## Make tasks available from every file

Zed language-extension tasks are associated with the active language. To make the same tasks available regardless of the active file, copy the checked task pack into the project:

```sh
mkdir -p .zed
cp /path/to/crabbox/integrations/zed/project-tasks.json .zed/tasks.json
```

If `.zed/tasks.json` already exists, merge the task objects instead of overwriting it.

## Typical workflow

1. Run **Crabbox: Doctor**.
2. Run **Crabbox: Spawn reusable box** and note the returned friendly slug.
3. Select a command such as `go test ./...` or `pnpm test`, then run **Crabbox: Run selected command** for an isolated one-shot run.
4. For repeated work on the warm box, use **Crabbox: Run command on box…** and enter the slug.
5. Use **Crabbox: Inspect box…** or **Crabbox: SSH into box…** when a failure needs live investigation.
6. Run **Crabbox: Stop box…** when finished.

## Security model

The extension stores no provider credentials, broker tokens, SSH keys, or lease state. It does not modify SSH configuration and does not call provider APIs directly. Interactive tasks pass the chosen lease id and command to the local Crabbox CLI, which remains the authoritative security and lifecycle boundary.

The interactive task helpers currently use Bash and therefore target macOS and Linux. Direct tasks such as Doctor, Spawn, Run selected command, Run detected project job, and List invoke `crabbox` directly and do not depend on Bash.
